🔹 Topics

  • 📌 A topic is a stream of data, similar to a table in a database
  • 🔠 Topics have names
  • ➕ You can create as many topics as needed

⬇️ Partitions

🔹 Partitions

  • 📂 Topics are divided into partitions
  • 🔢 Each message has an auto-incremented ID (offset)
  • ⚖️ Partitions don’t need to have the same length
  • 🛑 Immutable: Once written, messages can't change

🔹 Kafka Cluster

  • 🖧 Consists of multiple brokers
  • 🔌 Clients connect to the whole cluster via any broker

⬇️ Kafka Broker

🔹 Kafka Broker

  • 🖥️ A server that stores topic partitions
  • 📡 Each broker knows all topics & partitions
  • 🔄 Provides metadata to producers & consumers

⬇️ Topic Replication

🔹 Topic Replication

  • 🔁 Avoids data loss by replicating topics
  • 👑 Each partition has a leader
  • 🔄 Other brokers store replicated partitions

⬇️ Producers

🔹 Producers

  • ✉️ Send messages to Kafka topics
  • 🎯 Decide which partition to write to
  • ⚖️ Load balancing using round-robin

⬇️ Consumers

🔹 Consumers

  • 📥 Read data from topics
  • 📌 Read messages in order within partitions
  • 🔄 Can belong to consumer groups
